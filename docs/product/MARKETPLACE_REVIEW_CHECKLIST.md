# Marketplace Review Checklist

Use this when reviewing the live marketplace every week.

## Pages to check

- `/admin/analytics/explore-v2`
- `/admin/analytics/host`
- `/admin/requests`

## Record from Explore V2

- date range reviewed
- sheet opened
- primary CTR
- view details CTR
- trust cue rows: none, instant confirmation, unknown if present
- CTA copy rows: default, clarity, action, unknown if present
- strongest market
- strongest intent
- weakest market or intent

## Record from Host analytics

- guidance viewed
- fix clicks
- fix click CTR
- submit attempts
- improvement rate
- average score delta
- clicks by target step: basics, details, photos

## Record from Property Requests

- requests created
- requests published
- open requests
- requests with responses
- zero-response requests
- total responses sent
- response rate
- average first response
- median first response
- top 3 stall segments

## Keep / kill rules

### Explore V2 experiments

Keep a variant live if:

- opens are still low
- direction is mixed across weeks
- CTR gaps are small and unstable

Kill a variant if:

- it underperforms for two consecutive weekly reviews
- it is clearly worse across the main active segment, not just a tiny slice

### Host quality guidance

Iterate only if:

- one target step dominates clicks
- improvement rate is weak or flat
- average score delta is not moving meaningfully

### Property Requests

Intervene next on responder activation if:

- published requests rise
- zero-response requests stay high
- median first response stays slow

Do not broaden the product yet if:

- the current loop is not yet measured for at least a few weekly reviews
