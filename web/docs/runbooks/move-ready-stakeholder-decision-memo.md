## Decision logic

### Go

Only choose `go` if the validation scorecard is in the go range and operator burden stayed controlled.

### Iterate

Choose `iterate` if the wedge has real demand but routing, response speed, or operator handling still needs tightening.

### Pause/rework

Choose `pause/rework` if the wedge is not matching, not receiving provider responses, or is creating too much operational drag.

## What we are explicitly not doing next

- no tenant requester expansion
- no removals
- no public provider directory
- no scheduling
- no payments
- no reviews/ratings
- no monetisation rollout

## Recommended next batch

State one narrow follow-up only. If the scorecard does not pass, the next batch must be a wedge-hardening batch, not an expansion batch.
