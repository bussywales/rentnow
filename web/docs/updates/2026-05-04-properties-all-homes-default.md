## What changed

- Reordered the top category rail on `/properties` so `All homes` appears first.
- Updated the default `/properties` browse state so a plain visit opens on `All homes` instead of preselecting `To rent`.
- Explicit category and intent URLs still win, so direct links to rent, sale, short-let, and off-plan views keep working as before.

## Why it matters

- New visitors land on the broadest inventory view first.
- The first selected chip now matches the more neutral browse expectation for `/properties`.
- Users can still recover prior browse filters from the existing continue-browsing flow instead of having the base route silently inherit a rent-only default.
