## What changed

- Unified shortlet state wording so guest and host surfaces use explicit status language:
  - Request
  - Pending approval
  - Confirmed
  - Cancelled
  - Expired
- Added a one-line status banner in:
  - trip timeline cards
  - host bookings drawer
  - property detail booking widget

## Why this matters

- Removes ambiguous "pending" wording and reduces post-booking confusion.
- Keeps guest and host expectations aligned across booking, trip tracking, and host actions.
