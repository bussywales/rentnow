## What changed

- Added a dedicated host earnings timeline at `/host/earnings` with summary cards and tabs for `Available`, `Upcoming`, `Paid`, and `All`.
- Hosts can now see booking-level payout status with clear reasons when funds are not yet eligible.
- Added host actions per row:
  - `Open booking` for approval-related items
  - `Request payout` for eligible funds (manual pilot flow message)
  - `View details` for paid payouts
- Added a host top-nav entry (`Earnings`) and a host dashboard action-centre CTA when payout funds are available.

## Why this matters

- Hosts get a clear and trustworthy timeline of what they have earned and when payout is expected.
- Support and ops teams get fewer ambiguous payout questions thanks to explicit status and reason copy.
- Pilot payout operations remain safe and manual while still giving hosts visibility.

## Pilot note

- Pilot uses manual payouts. Automated payouts will be enabled later via admin setting.
