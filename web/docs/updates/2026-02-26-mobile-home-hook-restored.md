## What changed

- Reordered the mobile `/` discovery stack so **Featured homes** appears immediately below **Quick Start**.
- Kept discovery support rails below Featured homes for a listings-first hook.
- Gated optional rails:
  - Saved rail now renders only when there are saved items.
  - Recently viewed rail now renders only when there are viewed items.
- Retained the collapsed **Why PropatyHub?** section in the lower part of the mobile block so it does not interrupt the primary hook.

## Why this helps

- New and returning mobile users see real listings earlier.
- Less above-the-fold card clutter before first discovery actions.
- Better parity with listings-first mobile patterns.
