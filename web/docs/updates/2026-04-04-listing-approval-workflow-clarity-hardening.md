## What changed
- The host listings manager now keeps `changes requested` and `rejected` separate instead of collapsing them into broader draft or paused buckets.
- Each listing row now explains what the current approval state means in plain language and shows the next best action for that state.
- Stored moderation feedback is surfaced inline for `changes requested` and `rejected` listings when review notes already exist.

## Why it matters
- Paid landlords and agents can see faster whether a listing is still being reviewed, needs fixes, was rejected, or is simply paused.
- The path back to a live listing is clearer because the manager now points users to the right next step instead of showing a generic submit action.
- Support can tell more quickly whether a host is blocked by moderation feedback or by a non-approval issue such as billing or profile setup.
